# Contributing to NAAS-Agentic-Core

Thank you for your interest in contributing to the North African AI Safety Lab's open-source toolkit. We welcome contributions from researchers, developers, and safety advocates.

## Development Setup

To ensure a consistent development environment, please follow these steps:

1.  **Fork and Clone:**
    ```bash
    git clone https://github.com/YOUR-USERNAME/NAAS-Agentic-Core.git
    cd NAAS-Agentic-Core
    ```

2.  **Environment:**
    We recommend using Python 3.10+ and a virtual environment.
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows: venv\Scripts\activate
    pip install -r requirements.txt
    ```

3.  **Pre-Commit Hooks:**
    We use pre-commit hooks to enforce code style and quality.
    ```bash
    pip install pre-commit
    pre-commit install
    ```

## Code Style & Standards

*   **Python:** We follow PEP 8. Please run `black` and `flake8` before submitting.
*   **Documentation:** All new functions and classes must have docstrings (Google style).
*   **Testing:** New features must include unit tests in the `tests/` directory.

## Pull Request Checklist

Before submitting a PR, please verify:

- [ ] My code follows the style guidelines of this project.
- [ ] I have performed a self-review of my own code.
- [ ] I have commented my code, particularly in hard-to-understand areas.
- [ ] I have made corresponding changes to the documentation.
- [ ] My changes generate no new warnings.
- [ ] I have added tests that prove my fix is effective or that my feature works.
- [ ] New and existing unit tests pass locally.

## Contributing Evaluation Artifacts

If you are contributing to the evaluation suite (e.g., new prompts, safety definitions), please ensure you provide:

1.  **Suite Definition:** A JSON/YAML file defining the test cases (inputs).
2.  **Scoring Rules:** Clear criteria for what constitutes a "pass" or "fail" (e.g., specific keyword blocks, sentiment thresholds).
3.  **Sample Report:** An anonymised example of the output generated by your suite.
4.  **Language Coverage:** Explicitly state which languages/dialects are targeted (e.g., "Tunisian Darija mixed with French").
